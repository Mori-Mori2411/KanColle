# ファイル構造と役割の解説

## フォルダ構成

```
kancolle-manager/
├── index.html          # 画面のHTML構造・スクリプトの起点
├── css/
│   └── style.css       # 見た目（色・レイアウト・フォント）
├── js/
│   ├── firebase.js     # Firebase接続・Googleログイン
│   ├── resources.js    # 資材記録の保存・表示・CSV
│   └── expedition.js   # 遠征の追加・表示・理論値計算
└── README.md
```

ファイルを分けることで「どこを直せばいいか」がすぐわかります。
例えばログインの挙動を変えたければ `firebase.js` だけ見ればよく、
レイアウトを変えたければ `style.css` だけ触ればいいです。

---

## 各ファイルの役割

---

### index.html — 司令部

HTMLとJSをつなぐ「司令部」の役割です。
自分でデータ処理はせず、各JSファイルに仕事を任せます。

**やっていること：**
1. CSSとアイコンフォントを読み込む
2. 画面のHTML構造（ログイン画面・アプリ画面）を書く
3. JSファイルから関数を `import` して `window` に登録する
4. ログイン状態の監視（`onAuthStateChanged`）を起動する

**重要なポイント：**

```html
<script type="module">
  import { auth, signIn } from './js/firebase.js';
  import { saveRecord }   from './js/resources.js';
```

`type="module"` をつけることで `import` が使えます。
これがないと他のJSファイルを読み込めません。

```javascript
window.appSignIn = signIn;
```

`type="module"` 内の変数・関数は外から見えません。
HTMLの `onclick="appSignIn()"` から呼べるよう `window` に登録しています。

---

### css/style.css — 見た目担当

画面の見た目をすべてここで管理します。
JSやHTMLには色・サイズ・レイアウトのコードを書きません。

**重要なポイント：**

```css
:root {
  --bg:#fff;
  --text:#1a1a1a;
  --green:#1D9E75;
}
```

`:root` に色を変数として定義しています。
色を変えたいときはここの1行を書き換えるだけで全体に反映されます。

```css
@media(prefers-color-scheme:dark){
  :root{ --bg:#1e1e1c; --text:#e8e6df; }
}
```

OSのダークモード設定を検知して自動で切り替わります。
`var(--bg)` と書いておけばライト・ダーク両対応になります。

---

### js/firebase.js — Firebase接続担当

FirebaseとGoogleログインだけを担当します。
**設定値を差し替える以外、このファイルを編集することはほぼありません。**

**重要なポイント：**

```javascript
export { auth, db, signIn, signOut, onAuthStateChanged };
```

`export` した値は他のファイルから `import` して使えます。
`db`（データベース接続）を export しておくことで、
`resources.js` や `expedition.js` がデータを読み書きできます。

```javascript
const isConfigured = !firebaseConfig.apiKey.includes('YOUR_');
```

設定値が未記入のとき（`YOUR_API_KEY` のまま）に警告を出すための判定です。

---

### js/resources.js — 資材記録担当

資材のFirestore読み書き・画面表示・CSV出力を担当します。

**重要なポイント：**

```javascript
import { db } from './firebase.js';
```

`firebase.js` で export した `db` をここで使います。
ファイルを分けていても `import` で連携できます。

```javascript
export const RES = [
  { key:'fuel', label:'燃料', icon:'ti-flame', cap:350000, color:'#1D9E75' },
  ...
];
```

資材の定義をオブジェクトの配列にまとめています。
新しい資材を追加するときはここに1行追記するだけで
カード・入力欄・テーブル・CSVすべてに自動反映されます。

`expedition.js` でも資材の情報が必要なので `export` しておき、
そちらで `import { RES } from './resources.js'` して使っています。
同じデータを2ヶ所に書かずに済みます。

```javascript
export function listenRecords(uid) {
  const q = query(collection(db, `users/${uid}/records`), ...);
  unsubscribe = onSnapshot(q, snap => {
    records = snap.docs.map(d => ({ id: d.id, ...d.data() }));
    renderMetrics();
    renderLog();
  });
}
```

`onSnapshot` でFirestoreを監視し、変化があるたびに自動で画面を更新します。
`unsubscribe` に解除関数を保存しておき、ログアウト時に `stopListening()` で停止します。
停止しないとログアウト後もデータを読み続けてしまいます。

### js/expedition.js — 遠征管理担当

遠征の追加・削除・表示・理論値計算を担当します。

**重要なポイント：**

```javascript
import { RES_KEYS, RES } from './resources.js';
```

資材の定義を `resources.js` から借りて使っています。
同じ内容を2ファイルに書く必要がなくなります。

```javascript
async function saveExpeditions() {
  await setDoc(
    doc(db, `users/${currentUid}/data/expeditions`),
    expeditions
  );
}
```

遠征データはスロット（game/out/sleep）ごとの配列を
1つのオブジェクトにまとめて `setDoc` で上書き保存しています。
`addDoc`（追加）ではなく `setDoc`（上書き）を使う理由は、
削除や並び替えが発生するため「常に最新の全データで上書き」が
シンプルで管理しやすいからです。

```javascript
function buildTheoryHtml(slot, exps, hours) { ... }
function buildExpCardsHtml(exps, slot)      { ... }
```

HTMLを組み立てる処理を関数に分けています。
`renderSlot()` がこの2つを呼んで結合するだけにすることで、
「理論値だけ変えたい」「カードだけ変えたい」ときに
それぞれの関数だけ直せばよくなります。

## ファイル間の関係図

```
index.html
├── import → js/firebase.js    （auth, db, signIn, signOut）
├── import → js/resources.js   （listenRecords, saveRecord, ...）
└── import → js/expedition.js  （listenExpeditions, addExpedition, ...）

js/resources.js
└── import → js/firebase.js    （db）

js/expedition.js
├── import → js/firebase.js    （db）
└── import → js/resources.js   （RES, RES_KEYS）
```

`firebase.js` がすべての起点で、`db` を提供しています。
`resources.js` と `expedition.js` はお互いを直接呼び合わず、
`index.html` が両方の司令塔として調整しています。

---

## 新機能を追加するときの手順

例として「建造記録」機能を追加する場合：

1. `js/construction.js` を新規作成して保存・表示の関数を書く
2. `index.html` に建造記録のHTMLを追加する
3. `index.html` の `<script>` に `import` を追加する
4. `window.appSaveConstruction = ...` で `onclick` から呼べるようにする
5. `css/style.css` に見た目を追加する

各ファイルの役割が明確なので、追加・変更・削除の影響範囲が限定されます。
